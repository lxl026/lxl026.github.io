《C++ primer 第二章 变量和基本类型》 读书笔记

  • C++是一种静态数据类型语言，其类型检查发生在编译时。

  • 数据类型是程序的基础：它确定了数据的意义以及可对数据进行的操作。

  • 给无符号类型赋值一个超出其范围的值时，结果是初始值对无符号类型能表达的数值总数取模的结果。给符合类型赋值超范围时，结果是未定义的。

  • 切勿混用带符合类型和无符号类型。因为带符号类型或被先转换为无符号类型，一旦带符号类型为负数，结果将会不符合预期。

  • 声明使得名字为程序所知；
    定义负责创建与名字关联的实体。
    故变量能且只能被定义一次，但可以被声明多次。

  • 引用即别名，不是对象。定义引用时，程序把引用和初始值绑定（bind）在一起，一旦绑定完成，引用无法解绑，故引用必须初始化。

  • 0，NULL，和nullptr的区别
    在C语言中，NULL被预编译为((void*)0)，在C++中，NULL则预编译为0，这可能会引起函数重载的一些问题，如：
    void foo(type1 a, type2 *b);
    void foo(type1 a, int b);
    在调用foo(a,NULL)时，调用的是void foo(type1 a, int b)而不是我们预期的void foo(type1 a, type2 *b)。
    C++11中的nullptr 解决了这类问题。nullptr是一个std::nullptr_t类型的字面值空指针常量，它不会被转换为整型。

  • 在指针或引用的定义语句中=左边的*和&是类型修饰符，是声明符的一部分。如在int *p;中，*p是一个整体，int是其基本数据类型。
    int *p, a;中，p是指向int类型的指针，a是int类型。

  • const限定符
    const对象必须初始化，且初始化后不能再为之赋值。
    默认状态下，const对象仅在文件内有效，若要在文件间共享，则需要加extern关键字。

  • const引用
    非常量引用的类型必须与被引用的类型一致，但const引用则可以绑定非常量的对象，字面值甚至是一般表达式。在这个过程中实际上是产生了一个const类型的临时量，引用再去绑定这个临时量。如：
    执行double d = 3.14; const int &i = d;的实际过程为：double d = 3.14; const int tmp = d; const int &i = tmp;。

  • 指向常量的指针与常量指针
    const type *p = &val; p是一个指向常量的指针，可以改变它指向的对象，但是不能通过它修改对象的值。
    type *const p = &val; p则是一个常量指针，必须初始化，且初始化后不能修改它指向的对象，但是可以通过它修改对象的值。
    const type *const p = &val; p是一个指向常量的常量指针，既不能修改它指向的对象，也不能通过它修改对象的值。第一个const是底层const，修饰的是指针指向的对象，第二个const是顶层const，修饰的是指针本身。

  • 常量表达式：是值不会改变并且在编译过程就可以得到计算结果的表达式。

  • constexpr变量
    C++11允许将变量声明为constexpr类型，以便由编译器来验证变量的值是否是一个常量表达式。

  • 字面值类型（literal type）
    能用constexpr修饰的简单、易见、易得的类型，如算术类型，引用和指针。string类型、IO库和类类型不是字面值类型，不能用constexpr来修饰。

  • 指针与constexpr
    constexpr限定词只对指针有效，对指针指向的对象无效。
    const int *p = nullptr; // p是一个指向整型常量的指针
    constexpr int *q = nullptr // q是一个指向整数的常量指针

  • 类型别名

    • typedef double wages;
    • using INT32_t=int;C++11

  • auto类型说明符
    C++11新特性，可由编译器自动推断表达式类型，然后将变量设置为表达式类型。故auto定义的变量必须由初始化。
    同一语句中可定义多个变量，但变量的基础类型必须一致。
    auto一般会忽略顶层const而保留底层const

  • decltype类型指示符
    C++11新特性。在编译过程中推断表达式的返回类型，但不调用表达式，亦即不用表达式的结果初始化变量。
    decltype(f()) sum = x; // sum的类型就是f的返回值类型
    不同于auto，decltype不忽略顶层const。
    decltype的变量如果加上了一层或多层()，则会得到引用类型。
    指针解引用操作得到的结果是引用类型
