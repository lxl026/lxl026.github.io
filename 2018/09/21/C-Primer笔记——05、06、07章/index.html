《C++ Primer 第五、六、七章》读书笔记

  • 悬垂else
    else与离它最近的未匹配if匹配

  • switch内部的变量定义
    在switch内，如果某处一个带初值的变量位于作用域之外，在另一处该变量位于作用域之内，则从前一处跳转到后一处的行为是非法的。
    不要在case语句中定义变量，要定义的话就加上花括号，把变量局限在代码块内

  • 如果要用引用作为函数参数，尽量使用常量引用，否则容易带来误导，以及限制函数所接受的实参类型。

  • 尾置返回类型
    auto func(int i) -> int (*) [10] // 返回类型为指向长度为10的整型数组的指针

  • 重载与const
    顶层const不影响传入函数的对象，即void func(type)和void func(const type)是同一个函数
    底层const则不同，void func(const type *)和void func(type *)是两个函数

  • 重载与作用域
    局部函数声明会隐藏外层作用域中声明的同名函数实体

  • 在C++中，名字查找发生在类型检查之前

  • 内联函数可避免函数调用的开销
    内联说明只是向编译器发出一个请求，编译器可以忽略这个请求

  • constexpr函数
    其返回类型与形参必须都是字面值类型
    只有一个return语句
    若其实参不是常量，则其返回结果不是常量

  • assert预处理宏（#include <cassert>）
    assert(expr)
    若expr为真，assert什么也不做；若为假，则输出信息并终止程序的执行
    宏定义#define NDEBUG可解除assert

  • 函数匹配：
    候选函数：与被调用的函数同名，且其声明在调用点可见
    可行函数：形参与实参数量相等，且实参类型与对应的形参类型相同或可转换
    最佳匹配：有且只有一个函数，每个实参的匹配都不劣于其他可行函数，且至少有一个实参匹配优于其他可行函数提供的匹配

  • const成员函数
    const的作用是修改隐式this指针的类型
    不能在一个常量对象上调用普通的成员函数

  • 一般来说，如果非成员函数是类接口的组成部分，则这些函数的声明应该与类放在同一个头文件内

  • 只有类没有声明任何的构造函数时，编译器才会生成默认构造函数

  • 可变数据成员：mutable
    即使是const对象、从const成员函数也可以修改mutable变量

  • 当我们提供一个类内初始值时，我们必须以赋值符号=或花括号表示

  • 编译器处理完类中所有的声明后才会处理成员函数的定义

  • 类型名需要特殊处理
    类不能重新定义外层作用域中已定义的某个类型名，类中的类型名也要在使用前声明

  • 如果成员是const、引用或某种为提供默认构造函数的类类型，我们必须通过构造函数初始化列表为这些成员提供初始值

  • 成员的初始化顺序与他们在类中出现的顺序一致

  • 委托构造函数
    受委托构造函数先执行初始化列表，再执行函数体，然后把控制器交还给委托函数的函数体

  • 编译器只允许一步隐式的类类型转换

  • explicit抑制了构造函数参数的隐式类型转换
    它只对一个参数的构造函数有效，explicit关键字只能出现在类内，其构造函数只能用于直接初始化

  • 聚合类
    所有成员都是public的
    没有定义然后构造函数
    没有类内初始值
    没有基类也没有虚函数
    可以提供一个花括号括起来的成员初始值来初始化对象

  • 一般在类外定义和初始化静态成员

  • 静态成员和指针成员可以是不完全类型

  • 可使用静态成员作为默认参数
